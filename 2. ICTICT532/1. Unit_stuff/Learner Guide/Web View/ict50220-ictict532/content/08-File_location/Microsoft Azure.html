MS Azure is the cloud computing solution offered by Microsoft. Azure provides cloud infrastructure with more than 200 products that customers can use in their businesses. As other cloud services, Azure offers infrastructure, software, and platform as services. Azure is a service covering 140 countries with 55 geographical locations.

Azure architecture

Azure architecture consists of regions, availability zones and sets, resource groups and Azure resource manager.

Select the headings below to learn more about Microsoft Azure architecture.

Azure regions

Azure regions are locations which contain data centres that store your data, regions are located around the world. Each Azure region is paired with another region for fault tolerance.

Customers can choose the region where their data is to be saved. There could be government regulations on which region you can save data in. The cost of your Azure service will vary from region to region.

Availability zones and sets

Availability zone is a physical location with one or more data centres. Each region has at least three availability zones. The purpose of availability zones is to achieve high availability. If one data centre is not working, the other two data centres can keep the region working.

Availability sets help to achieve availability and provide application redundancy across multiple virtual machines. You can either use availability zones or availability sets but not both.

Resource groups

Allow customers to organise and manage resources for a particular application, you can add virtual machines, SQL databases, network interfaces and other resources related to a particular application under one resource group.

Used to store huge amounts of unstructured data. Customers can store any type of data and access them as and when they want. Blob storage can meet any capacity requirements for a lower price.

Azure blob storage is durable, scalable, and secure. It uses role based access control with active directory to make sure your data is secure. Azure can create Data Lakes using blob storage for analytical purposes.
