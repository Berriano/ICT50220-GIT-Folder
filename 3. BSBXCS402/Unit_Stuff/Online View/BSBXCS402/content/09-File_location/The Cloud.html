Cloud computing is a concept that helps organisations to ease the burden of managing IT infrastructure and save money on capital expenditure spent on infrastructure. Today's organizations can implement their complete IT network and run their applications on the cloud.

A simple cloud solution for a micro or small business could be services such as OneDrive. However, for a more secure option it would be preferrable to look at cloud services offered by the numerous vendors currently in the marketplace.

Note

For this unit we will be only looking at cloud as a storage service, although the basics will still be covered.

Infrastructure as a Service (IaaS)

Hardware based IT resources in a network can be categorised as infrastructure, such as Computers (servers), Storage, Networking resources such as routers and switches. Cloud can host these resources in a virtual environment where consumers can implement their complete network in the cloud which is called Infrastructure as a Service.

IaaS service offered in cloud is not a pre-configured service; consumers have full control over how they want the network and infrastructure deployed, therefore the responsibility of managing the cloud is placed directly upon cloud consumers.

Platform as a Service (PaaS)

Platform as a service includes preconfigured environment of IaaS, cloud consumers only use the service as a platform to run their development tools, use operating systems, and for business analytics.

In this model burden of infrastructure management is taken away from the consumers and consumers only pay for the platform services they use.

Software as a Service (SaaS)

Software as a service let consumers to use software programs in the cloud without installing them in local computers. SaaS already includes IaaS and PaaS, but consumers only use and pay for SaaS built on IaaS and PaaS.

Best example for SaaS is Microsoft Office 365. Consumers can use complete Microsoft office package online by subscribing to MS Office 365 and pay for monthly subscription.

There are three main cloud computing models, distinguished by ownership, size and access.

Select the tabs to learn more about these:

Public Cloud

Public cloud is hosted by a provider, available to all and connected via an internet connection. Public cloud services are offered to consumers at a cost. Cloud provider is responsible for physical infrastructure and maintenance of their resources.

Consumers need the least technical expertise to use public cloud compared to other cloud computing models.

Private Cloud

Private cloud is owned and hosted by private organization (on-premises), with the purpose of centralizing access to different departments or sections of the organization. The access to private cloud is only for people who have permission to do so. Actual administration of the private cloud could be under internal staff or out soured company. Private cloud may physically be in the same location, but the cloud services should be available to access remotely to cloud consumers.

Hybrid Cloud

Hybrid cloud consists of aspects of both private cloud and public cloud. For example, organization may have their server running on premises but wanted to create storage in a cloud that is connected and synchronised to local server. For another example, an organisation may prefer to store their sensitive data on-premises while running other services on the cloud. Most important configuration concern for hybrid cloud would be that of the integration between public and private cloud.

Note

Each type of cloud offers pros and cons for their type. The public cloud although is the cheapest to run as it is an operational expense for a business, lacks security, but also requires the least amount of IT expertise. Whereas the private cloud can have the highest security, but also requires the highest IT expertise as everything is managed by the business.
