Network security

Data security is not just about having encryption on the data during transmission, but also restricting the access to the location of the data. There are a number of areas where security to a system can be improved:

Passwords. This is the simplest form of security for any system, it is also the easiest area for a hacker to break through due to weak strength of some password policies. Password strength not only applies to when you login to a device or network, but also into applications that access personal information for clients and yourself.

Strength denotes the general length of the password as well as the combination of potential character combinations you select, such as symbols, numbers, and lower or upper-case letters.

Select each of the tabs to learn more.

  • Strong

A weak password may only consist of 6 characters, mainly lower-case and maybe some numbers, with an example being John63. Dates, names, teams, anniversaries, vehicle registrations, pets and places are frequently used by people for passwords, and so they are also the options attempted first by attackers. 'Password' is still unfortunately the most commonly used password.

A strong password is a longer and more haphazard mixture of characters. With the strongest passwords considered as passphrases. Comparable to a password, but instead of creating a string of letters, numbers, and symbols, use words that make up a silly statement. For example: aCorgimake5agreaTh@mburg3r! This is an extremely strong password as who would have thought a corgi can make a great hamburger.

Other security that needs to be implemented on a network:

  • A properly administered firewall
  • Security suite that is industry recognised and gives end-point protection
  • Two-factor authentication for accessing sensitive information or applications
  • Good staff training in email protocol, especially in recognising phishing.
