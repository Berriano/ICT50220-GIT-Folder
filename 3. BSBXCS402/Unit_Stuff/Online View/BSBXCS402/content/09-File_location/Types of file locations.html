In a small business that location on sensitive files would normally be in the location of the businesses data files. Although the location would be the same location, the folder structure would differ in having folders with sensitive data being properly secured.

Let us have a look at a few scenarios in ways that a small business stores their data, keeping in mind that all business models will normally have 3 options – onsite storage, cloud storage, or a mixture of both.

Select the tabs below to learn more about types of file locations:

Micro business with standalone computer

This type of business might be a home-based business that does consultancy work, or a business that have a number of service staff but having a single computer that managers and plans all the business activities.

Data capacity 10GB to 250GB

The structure of the business could be:

  • Bookkeeping application running straight from the vendor's cloud
  • Client contacts running through cloud-based application such as Microsoft Outlook
  • File storage can be located on the computer or cloud, or synchronised with the cloud

Small network business up to 10 users

This type of business could have a small 5 to 10 user peer-to-peer or server-client network. These types of businesses can range from a small motel complex, retail outlet, small medical clinics, to small engineering firms.

Data capacity 250GB to 1TB

The structure of the business could be:

  • Bookkeeping application running straight from the vendor's cloud
  • Client contacts running through cloud-based application such as Microsoft Outlook
  • File storage can be located on a network attached storage (NAS) device, or server, or cloud, or synchronised with the cloud

Small network business up to 50 users

This type of business would have a server-client network. These types of businesses can range from a small business with multiple departs, to design centres or application development hubs.

Data capacity 1TB and above.

The structure of the business could be:

  • Accounting application running straight from the vendor's cloud, or server-based
  • Client contacts running through a cloud-based or server-based application such as a customer relationship management system
  • File storage managed through the server via NAS, synchronised with the cloud

Note

As you can see from the above scenarios the way that files can be stored can easily vary. This also means that the way that businesses secure sensitive information will also vary depending on where the information is stored.

744358555 / H_Ko / shutterstock.com

Less than 10 years ago, a business file storage location would have been on a hard disk drive (HDD). On a standalone system this would be an internal HDD with backups to external USB memory stick, or portable HDD, depending on the amount of data.

A small network (5 computers in a peer-to-peer configuration) would designate one computer as the main file storage device with all other computers having mapped folders linked to them, or a network attached storage (NAS) device and all computers mapped to it.

On a server/client network (domain), the server would have 2 or more HDD installed for the main storage. These drives would be attached to a RAID controller for redundancy and backups would be performed to a tape cartridge drive embedded in the server.

Step forward to today, and with businesses have access to the NBN in most places, cloud storage is now becoming more and more the norm.
