There are three main cloud computing models, distinguished by ownership, size and access.

— 699701578 / Bakhtiar Zein / shutterstock.com. Modified by TAFE Queensland

Private cloud

Private cloud is owned and hosted by a private organisation (on-premises), with the purpose of centralising access to different departments or sections of the organisation. The access to private cloud is only for people who have permission to do so. Actual administration of the private cloud could be under internal staff or an outsourced company. Private cloud may physically be in the same location but the cloud services should be available to access remotely to cloud consumers.

Public cloud

Public cloud is hosted by a provider, available to all and connected via an internet connection. Public cloud services are offered to consumers at a cost. The cloud provider is responsible for the physical infrastructure and maintenance of their resources. Consumers need the least technical expertise to use public cloud compared to other cloud computing models.

Examples are Microsoft Azure, Amazon Web Services and Google Cloud. These are the leading public cloud solutions in the industry.

Hybrid cloud

Hybrid cloud consists of aspects of both the private cloud and the public cloud. For example, an organisation may have their server running on the premises but wanted to install a database in a cloud that is connected to a local server. For another example, an organisation may prefer to store their sensitive data on-premises while running other services on the cloud.

The most important configuration concern for hybrid cloud would be the integration between public and private cloud.

Each type of cloud offers pros and cons for their type. The public cloud is the cheapest to run as it is an operational expense for a business, lacks security, but also requires the least amount of IT expertise. Whereas, the private cloud can have the highest security, but also requires the highest IT expertise as everything is managed by the business.
